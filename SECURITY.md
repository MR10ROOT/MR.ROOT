## 🐛 Reporting a Vulnerability

If you discover a security vulnerability **in this tool itself** (e.g. path traversal in report saving, command injection in input handling, insecure temp file usage), please report it responsibly.

**Do NOT open a public GitHub Issue for security vulnerabilities.**

### How to report:
1. Open a **private** GitHub Security Advisory:  
   `GitHub repo → Security tab → Report a vulnerability`
2. Or contact via the details on [github.com/MR10ROOT](https://github.com/MR10ROOT)

### What to include:
- Description of the vulnerability
- Steps to reproduce
- Affected module / version
- Potential impact

### Response time:
- Acknowledgment: within **72 hours**
- Fix / patch: within **14 days** for critical issues
